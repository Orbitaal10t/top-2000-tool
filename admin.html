Top 2000 Tool - Setup Instructies

Leer hoe je de Spotify Client ID configureert voor je eigen deployment van deze app.

Let op!
Deze instructies zijn alleen nodig als je je eigen versie van deze app host. Als je de app gebruikt via de standaard URL, is alles al geconfigureerd!

1 Maak een Spotify Developer App

  1. Ga naar de Spotify Developer Dashboard
  2. Log in met je Spotify account
  3. Klik op "Create app"
  4. Vul de volgende gegevens in:
    • App name: Top 2000 Tool
    • App description: NPO Radio 2 Top 2000 tracker
    • Redirect URI: De URL waar je app draait + /index.html
      Bijvoorbeeld: https://jouwdomein.nl/index.html
    • API: Selecteer "Web API"
  5. Accepteer de voorwaarden en klik op "Save"
  6. Kopieer je Client ID (je vindt deze op de app detail pagina)

3 Deploy de App

Deployment gebeurt automatisch via GitHub Actions.

  1. Push je wijzigingen naar de main branch
  2. GitHub Actions start automatisch de deployment
  3. Bestanden worden via SSH/rsync naar je server gesynchroniseerd
  4. Je app is nu klaar voor gebruik!

Hoe werkt het?

De app laadt de configuratie uit config.js: 

const CONFIG = {
    SPOTIFY_CLIENT_ID: 'jouw-client-id-hier',
    NPO_API_URL: 'https://www.nporadio2.nl/api/...',
    // ... andere instellingen
};

De app controleert eerst of config.js beschikbaar is. Als deze ontbreekt of geen geldige Client ID bevat, toont de app automatisch de Custom Client Setup wizard waar gebruikers hun eigen Client ID kunnen invoeren.

Veiligheid

  • Client ID is publiek: De Spotify Client ID is geen geheim - het is veilig om deze in je frontend code te hebben
  • PKCE Flow: Deze app gebruikt PKCE (Proof Key for Code Exchange) voor veilige authenticatie zonder Client Secret
  • Geen geheimen nodig: PKCE maakt gebruik van dynamisch gegenereerde codes per authenticatie sessie
  • Beveiliging door design: Spotify's OAuth 2.0 PKCE flow is specifiek ontworpen voor public clients zoals web apps

Veelgestelde Vragen

Waarom zie ik de melding "Spotify is nog niet geconfigureerd"?

Dit betekent dat config.js niet bestaat of geen geldige Client ID bevat. Volg de configuratie-instructies hierboven om dit op te lossen.

Kan ik de app lokaal testen?

Ja! Voor lokale testing kun je de "Custom Client Setup" functie gebruiken om tijdelijk je Client ID in te voeren zonder config.js te maken. Of maak een config.js bestand aan (dit wordt niet gecommit dankzij .gitignore).

Is het veilig om mijn Client ID in de repository te committen?

Ja, absoluut! De Spotify Client ID is expliciet ontworpen om publiek te zijn. Spotify gebruikt PKCE voor beveiliging, wat betekent dat er geen Client Secret nodig is. Elke gebruiker die je website bezoekt kan de Client ID toch in de browser bekijken.

Moet ik mijn Spotify app in Production Mode zetten?

Nee, Development Mode is voldoende voor persoonlijk gebruik. Production Mode is alleen nodig als je meer dan 25 gebruikers hebt.
