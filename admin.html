Top 2000 Tool - Setup Instructies

Leer hoe je de Spotify Client ID configureert voor je eigen deployment van deze app.

Let op!
Deze instructies zijn alleen nodig als je je eigen versie van deze app host. Als je de app gebruikt via de standaard URL, is alles al geconfigureerd!

1 Maak een Spotify Developer App

  1. Ga naar de Spotify Developer Dashboard
  2. Log in met je Spotify account
  3. Klik op "Create app"
  4. Vul de volgende gegevens in:
    • App name: Top 2000 Tool
    • App description: NPO Radio 2 Top 2000 tracker
    • Redirect URI: De URL waar je app draait + /index.html
      Bijvoorbeeld: https://jouwdomein.nl/index.html
    • API: Selecteer "Web API"
  5. Accepteer de voorwaarden en klik op "Save"
  6. Kopieer je Client ID (je vindt deze op de app detail pagina)

2 Configureer GitHub Secret

  1. Ga naar je GitHub repository
  2. Klik op "Settings" → "Secrets and variables" → "Actions"
  3. Klik op "New repository secret"
  4. Vul in:
    • Name: SPOTIFY_CLIENT_ID
    • Secret: Plak hier je Spotify Client ID
  5. Klik op "Add secret"

3 Deploy de App

De Spotify Client ID wordt automatisch geïnjecteerd tijdens deployment via GitHub Actions.

  1. Push je wijzigingen naar de main branch
  2. GitHub Actions start automatisch de deployment
  3. De Client ID wordt vervangen in index.html voordat de bestanden worden gedeployed
  4. Je app is nu klaar voor gebruik!

Hoe werkt het?

De deployment workflow in .github/workflows/deploy.yml voert de volgende stap uit: 

- name: Inject Spotify Client ID
  run: |
    sed -i "s/SPOTIFY_CLIENT_ID_PLACEHOLDER/${{ secrets.SPOTIFY_CLIENT_ID }}/g" index.html

Dit vervangt de placeholder SPOTIFY_CLIENT_ID_PLACEHOLDER in de code met de echte Client ID uit je GitHub Secret. Zo blijft je Client ID veilig en werkt de app overal waar hij gehost wordt!

Veiligheid

  • Client ID is publiek: De Spotify Client ID is geen geheim - het is veilig om deze in je frontend code te hebben
  • PKCE Flow: Deze app gebruikt PKCE (Proof Key for Code Exchange) voor veilige authenticatie zonder Client Secret
  • Geen secrets in repository: De Client ID wordt nooit gecommit naar je repository
  • GitHub Secrets: Je Client ID wordt veilig opgeslagen in GitHub en alleen gebruikt tijdens deployment

Veelgestelde Vragen

Waarom zie ik de melding "Spotify is nog niet geconfigureerd"?

Dit betekent dat de SPOTIFY_CLIENT_ID GitHub Secret nog niet is ingesteld of dat de app nog niet is gedeployed na het instellen van de secret.

Kan ik de app lokaal testen?

Ja! Open index.html in een browser en gebruik de "Custom Client Setup" functie om tijdelijk je eigen Client ID in te voeren voor local testing.

Moet ik mijn Spotify app in Production Mode zetten?

Nee, Development Mode is voldoende voor persoonlijk gebruik. Production Mode is alleen nodig als je meer dan 25 gebruikers hebt.
